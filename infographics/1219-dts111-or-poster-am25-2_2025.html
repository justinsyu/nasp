Exchanging Genomics Reports Between Pathology Labs and Medical Centers Using the mCODE FHIR Implementation Guide

A CodeX pilot tested whether molecular pathology reports from Tempus could be transformed into mCODE-conformant FHIR files and decoded by academic cancer centers for clinical use and standards refinement.

Method

What the mapping had to carry

mCODE was used as the exchange layer for core oncology report content, including operational, specimen, molecular, and therapeutic implication fields.

Lab and order context Facility, order information, and pathologist metadata.
Patient demographics Report recipient context needed for EHR integration.
Specimen details Tumor report source material and related identifiers.
Molecular studies Discrete genomic findings formatted for machine exchange.
Therapeutic implications Treatment insight fields such as medication relevance.
Trial eligibility Oncology decision support content beyond legacy EHR standards.
Syntax alignment Nominal report changes, including drug class concatenation.
Standards feedback Compatibility refinements routed back through HL7/CodeX.

Results

Most work centered on normalization, not rebuilding the report

>50 elements mapped directly

Genomic data elements from Tempus reports aligned to mCODE fields.

Some annotations adjusted

Required changes included added genomic annotations and syntax adjustments.

2 sites received files

Vanderbilt and CHLA decoded transmitted mCODE files for downstream use.

Conclusion

mCODE can support practical genomic data exchange

01

mCODE aligned molecular report exchange with HL7 standards and lightweight FHIR transmission.

02

Only nominal changes were needed to existing pathology reports for compatibility.

03

Cross-domain collaboration between pathology labs and academic centers was central to adoption.
