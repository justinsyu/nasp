Creating a Digital Refill Program Using Readily Available Software

Intermountain Specialty Pharmacy built a refill intake channel from existing health-system tools, then tracked whether patients would repeatedly use it and whether the pharmacy could grow dispenses without adding phone time.

Objective

Make digital refill setup possible with tools already on hand.

The project replaced a phone-only refill setup model with a patient-facing digital questionnaire and a back-office work queue. The evaluation focused on adoption, repeat use, call burden, and whether the program supported a higher dispense volume.

Methods

A low-code refill pathway turned texts and QR magnets into a work queue.

F

Create the refill form

A Microsoft Forms questionnaire captured refill request details from patients.

S

Send the link

Staff manually sent the form link by SMS through SureScripts Patient Communications Solution in EnterpriseRx.

Q

Add passive access

QR code magnets with the same link gave patients another way to reach the refill questionnaire.

L

Queue submissions

Microsoft Power Automate added each submitted refill request to a Microsoft List for order setup.

Results over time

Digital requests climbed while phone work per dispense moved down.

Text refill requests Phone calls per dispense Average phone minutes per dispense
Demand shifted quickly. Quarterly text refill requests increased from 500 in 3Q21 to 9,000 in 2Q23, exceeding the poster's more-than-10-fold growth statement.
Dispense volume rose. Specialty dispenses increased from 17,500 to 22,000 across the same period.
Phone intensity decreased. Calls per specialty dispense fell from 2.5 to 1.7, and average phone minutes per dispense fell from 7.0 to 4.9.

Why it mattered

  • Phone-only refill setup can create long hold times and lower patient satisfaction.
  • Patients of all ages prefer digital communication with healthcare providers.
  • Not every specialty pharmacy has access to bidirectional patient messaging.

What worked

  • 5,722 patients used the digital refill program.
  • Half of those patients used the program 5 or more times.
  • The pharmacy reached a current state where 40% of refills use the digital route.

What still needed calls

  • Some submitted information was inaccurate.
  • Some patients requested delivery dates that were not possible.
  • Those exceptions still required telephonic follow-up.

Conclusion

More refill volume, without matching growth in active phone time.

The digital refill program let the pharmacy absorb more specialty dispenses while inbound and outbound call volumes stayed broadly similar across the observed quarters.
